## 断言
```rust
fn main() {
    let a = 1;
    let b = 2;
    assert!(a == b, "{} was not equal to {}", a, b); // 不为true 则输出后续错误信息，并停止进程
    assert_eq!(a,b, "{} and {} are not equal", a, b ); // 对比两个值，如果不相等，也是同样处理
    debug_assert!(a == b , "debug assert {} and {} are not equal", a, b ) // 在debug模式下才生效
}
```

## 单元测试
```rust
#main.rs
fn main() {
        print!("main!!!");    // 由于是跑下面的测试代码，而且还没有调用这里，所以它是不会执行的。
}

```
```bash
# 单个测试文件的编译测试命令
rustc --test main.rs     #会进行编译test内容，而且会生成一个main 的可执行文件， 直接执行即可。
# 项目
cargo test   #运行项目中全部的测试
```
```rust
// lib.rs 
// 动态库的测试，跟上面一样
fn sum(a:i8, b:i8) -> i8{
    a + b 
}

#[cfg(test)]
mod tests {
    fn sum_inputs_outputs() -> Vec<((i8, i8), i8)>{
        vec![((1,1),2),((0,0),0),((2,-2),0 )]
    }
    #[test]
    fn test_sums(){
        for(input,output) in sum_inputs_outputs(){
            assert_eq!(crate::sum(input.0, input.1),output);
        }
    }

    #[test]
    #[should_panic] // 表示运行 this_panics 函数应该导致不可恢复的故障，在 Rust 中这类异常被称为 panic。
    fn this_panics(){
        assert_eq!(1,2);
    }
        
    #[test]
    #[ignore]  // 表示在执行 test的时候会跳过这个测试内容，只能使用特定的参数调用
    pub fn test_silly_loop(){
        super::silly_loop();
    }
}
```

## 集成测试
和正常测试的区别就是将测试代码都放入了根目录下的 tests这个测试目录下，并且对外开放，而且该测试程序还会被单独的编译成一个单独的可执行文件。
```rust
// /tests/sum.rs
use integration_test::sum;

#[test]
fn sum_test(){
    assert_eq!(sum(6,8), 14);
}
```
```rust
// /src/lib.rs  
// 必须是对外开放的pub，不然外部是无法调用和测试的
pub fn sum(s1 :u64, s2: u64) -> u64 {
    s1 + s2
}
```

## 共享通用代码
下面代码的编译方式很特殊，如果指定运行测试 test_with_fixture 函数的话，那么测试会隐藏或者捕获 printIn!宏，不会输出到测试工具中， 需要使用特定的参数才能让他输出：`cargo test test_with_fixture -- --nocapture` ，其中 -- 是cargo自身参数结束的标记，后续的任何参数都是传递给cargo调用的二进制文件的(libtest)， --nocapture 是传递给测试运行器的。相当于在编译完成后的调用： `./target/debug/包名-哈希值 test_with_fixture --nocapture`
```rust
mod common;
use common::{setup, teardown};

#[test]
fn test_with_fixture(){
    setup();
    assert_eq!(sum(7, 14), 21);
    teardown();
}
```
```rust
// common.rs
pub  fn setup(){
    println!("Setting up fixtures");
}

pub fn teardown(){
    println!("Tearing down");
}
```

## 基准测试工具
运行基准测试的命令： `cargo bench`
```rust
#![feature(test)]
extern crate test;

use test::Bencher;

pub fn do_nothing_slowly(){
    print!(".");
    for _ in 1..10_000_000 {};
}

pub fn do_nothing_fast(){
}

#[bench]
fn bench_nothing_slowly(b: &mut Bencher){
    b.iter(|| do_nothing_slowly());
}

#[bench]
fn bench_nothing_fast(b:&mut Bencher){
    b.iter(|| do_nothing_fast());
}
```
